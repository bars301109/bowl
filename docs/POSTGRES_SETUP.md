# 🐘 Бесплатное решение для сохранения данных на Render

## Проблема
При каждом редеплое на Render (бесплатный план) все данные теряются, потому что файловая система эфемерная.

## Решение: PostgreSQL (БЕСПЛАТНО!)

Render предоставляет **бесплатный PostgreSQL** до 256 MB, что достаточно для вашего проекта.

## Быстрая настройка (5 минут)

### Шаг 1: Создайте PostgreSQL базу данных на Render

1. Зайдите на https://dashboard.render.com
2. Нажмите **"New +"** → **"PostgreSQL"**
3. Заполните:
   - **Name**: `akylman-db` (или любое имя)
   - **Database**: `akylman` (или любое имя)
   - **User**: `akylman` (или любое имя)
   - Остальное оставьте по умолчанию
4. Нажмите **"Create Database"**
5. Подождите ~2 минуты пока база создастся

### Шаг 2: Скопируйте строку подключения

1. После создания базы, откройте её страницу
2. Найдите **"External Database URL"** (⚠️ Используйте External, не Internal!)
3. Скопируйте всю строку подключения
   - Выглядит примерно так: `postgresql://user:password@dpg-xxx-a.oregon-postgres.render.com:5432/database?sslmode=require`
   - **Важно**: Должен содержать `.render.com` в домене и параметр `?sslmode=require`

### Шаг 3: Добавьте переменную окружения

1. Вернитесь на страницу вашего сервиса **akylman-quiz**
2. Откройте вкладку **"Environment"** в левом меню
3. Нажмите **"Add Environment Variable"**
4. Добавьте:
   - **Key**: `DATABASE_URL`
   - **Value**: Вставьте скопированную строку подключения из Шага 2
5. Нажмите **"Add"**
6. Прокрутите вниз и нажмите **"Save Changes"**

### Шаг 4: Установите зависимости (если еще не установлены)

Код уже обновлен для поддержки PostgreSQL. Просто убедитесь, что в `backend/package.json` есть:

```json
"pg": "^8.11.3"
```

Если нет, добавьте в терминале:
```bash
cd backend
npm install pg
```

### Шаг 5: Задеплойте

1. После сохранения переменных окружения, нажмите **"Manual Deploy"**
2. Или сделайте `git push origin main`
3. Дождитесь завершения деплоя

### Шаг 6: Проверьте логи

После деплоя проверьте логи. Должно быть:

```
✅ Akylman Quiz Bowl Server Started
✅ PERSISTENT STORAGE MODE - Using PostgreSQL
✓ Database: PostgreSQL (managed by Render)
✓ Data will persist across redeploys
```

Если видите это - всё работает! 🎉

## Как это работает

- **Без DATABASE_URL**: Приложение использует SQLite (локальный файл, данные теряются)
- **С DATABASE_URL**: Приложение автоматически переключается на PostgreSQL (данные сохраняются)

## Что сохраняется

✅ Все команды (teams)  
✅ Все тесты (tests)  
✅ Все результаты (results)  
✅ Все категории (categories)  
✅ Все настройки (settings)  
✅ Все вопросы (questions)  

## Лимиты бесплатного плана

- **Хранилище**: 256 MB (достаточно для ~10,000 команд и результатов)
- **Подключения**: 10 одновременных
- **RAM**: 512 MB
- **CPU**: Общий

Этих лимитов достаточно для школьного/университетского соревнования.

## Мониторинг базы данных

1. Зайдите на страницу вашей PostgreSQL базы на Render
2. Прокрутите вниз, чтобы увидеть:
   - Размер базы данных
   - Количество подключений
   - Использование ресурсов

## Если нужно больше места

1. Render Dashboard → Ваша PostgreSQL база → "Change Plan"
2. Обновите до **Standard** ($15/месяц) или выше

## Устранение неполадок

### "cannot connect to database" или "getaddrinfo ENOTFOUND"
- **Используйте External Database URL** (не Internal) из дашборда PostgreSQL
- URL должен содержать домен `.render.com` (например, `dpg-xxx-a.oregon-postgres.render.com`)
- URL должен заканчиваться на `?sslmode=require`
- Проверьте, что в Environment Variables вашего сервиса есть DATABASE_URL
- См. [POSTGRES_FIX.md](POSTGRES_FIX.md) для подробной инструкции по исправлению
- Передеплойте после исправления

### "database pool error"
- PostgreSQL может еще запускаться, подождите несколько минут
- Попробуйте передеплоить

### Нужно сбросить базу данных

Удалить все данные и начать заново:

1. Зайдите на страницу вашей PostgreSQL базы на Render
2. Прокрутите вниз до **"Danger Zone"**
3. Нажмите **"Reset Database"**
4. Передеплойте ваш сервис

## Готово!

Теперь ваши данные будут сохраняться при каждом редеплое! 🎉

## Для разработчиков

Приложение автоматически определяет тип базы данных:
- Если `DATABASE_URL` установлен → PostgreSQL
- Если нет → SQLite (для локальной разработки)

Это позволяет:
- ✅ Работать локально без настройки базы
- ✅ Масштабироваться в продакшене с PostgreSQL
- ✅ Нулевая потеря данных после редеплоя
